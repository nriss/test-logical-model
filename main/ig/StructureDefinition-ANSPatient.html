Logical Model: Modèle Patient ANS (extension EHDSPatient)

Official URL: https://interop.esante.gouv.fr/ig/fhir/test-logical-model/StructureDefinition/ANSPatient Version: 0.1.0
Draft as of 2026-03-27 Computable Name: ANSPatient

Extension du modèle EHDSPatient pour le contexte français. Traduit en français les libellés des éléments hérités et ajoute des champs spécifiques au système de santé français.

Utilisations:

  • Ce Modèle logique n'est utilisé par aucun autre profil dans ce guide d'implémentation

Vous pouvez également vérifier les usages dans le FHIR IG Statistics

ANSPatient 0..* EHDSPatient Modèle Patient ANS (extension EHDSPatient)

Éléments définis dans les ancêtres :identifier, name, dateOfBirth, deceased[x], administrativeGender, address, telecom
Les instances de ce modèle logique peuvent être la cible d'une référence
identifier 1..* Identifier Identifiant du patient
name 0..* EHDSHumanName Nom du patient
dateOfBirth 0..1 dateTime Date de naissance
deceased[x] 0..1 Patient décédé / Date de décès
deceasedBoolean boolean
deceasedDateTime dateTime
administrativeGender 0..1 CodeableConcept Genre administratif
Description du binding: (preferred): HL7 Administrative Gender
address 0..* EHDSAddress Adresse(s)
telecom 0..* EHDSTelecom Coordonnées de contact
insFichier 0..1 Identifier INS-NIR (Identifiant National de Santé - Numéro au Fichier)
insNia 0..1 Identifier INS-NIA (Identifiant National de Santé - Numéro d'Identification d'Attente)
lieuDeNaissance 0..1 CodeableConcept Lieu de naissance
nationalite 0..1 CodeableConcept Nationalité
situationFamiliale 0..1 CodeableConcept Situation familiale
